crackingPasswordAPPDec Dec 2017 – May 2018

Our first approach to this project was to borrow CPU cycles from visitor's web browsers to crack MD5 password hashes. The worker creates an instance of the node class using socket.io with properties to track the progress of each worker and the chunk of passwords that was distributed. We have a list of passwords and a list of hashed passwords in a file where each worker gets a chunk of passwords to decrypt. We assigned a range of passwords(from the passwords.txt file) based on the number of workers that are connected to the server. We use a set of rules to crack each hashed password. We are using parallel computing when distributing the list of passwords within the workers by chunks. Instead of one worker (cpu) cracking all the passwords, we distribute the list of password within a group of workers. This decreases the time it takes to complete the entire list of passwords. Their browser will receive MD5 hashes and password candidates from a command-and-control server and report back any passwords it cracks for the duration of the time the visitor is browsing the "infected" website.More info.
